# 🚀 Crudstack

> **Build type-safe, database-agnostic CRUD applications in Next.js.**

Crudstack is a modern, monorepo-based toolkit designed to accelerate the development of admin dashboards and CRUD applications. By leveraging the **Adapter Pattern**, Crudstack decouples your business logic from your database, allowing you to seamlessly swap between Sanity, Firebase, Supabase, or Prisma without changing a single line of your UI or service code.

## ✨ Key Features

- **🔌 Database Agnostic**: Switch databases by simply changing the adapter. Core logic remains untouched.
- **🛡️ 100% Type-Safe**: Built with TypeScript, Zod, and strict generic interfaces.
- **🎨 Beautiful UI**: Pre-configured with `shadcn/ui`, Tailwind CSS, and ready-to-use layouts, data tables, and form views.
- **📦 Monorepo Architecture**: Powered by `pnpm` workspaces for optimal dependency management and lightning-fast builds.
- **⚡ CLI Scaffolding**: Generate a fully configured Next.js app in seconds with `pnpm create crudstack-app`.

## 💡 Key Architectural Highlights

- **Zero Vendor Lock-in**: Because of the Adapter Pattern, switching from Firebase to Sanity requires changing exactly **one line of code** in your `lib/crudstack.ts` file. Your UI, forms, and server actions remain 100% untouched.
- **Unified Context**: No more wrapping your app in 5 different providers. One `CrudStack` instance handles both your Database and Auth, accessible anywhere via the `useCrudStack()` hook.
- **Factory Pattern for DB**: Creating a service for a new collection is as simple as `crudstack.db<MyType>("collection_name")`. Full type inference, zero boilerplate.

## 🏗️ Architecture

Crudstack is organized as a `pnpm` workspace monorepo:

```bash
crudstack/
├── templates/
│   └── nextjs/          # The base Next.js app generated by the CLI
├── packages/
│   ├── core/             # @crudstack/core (Interfaces & Generic Provider)
│   ├── adapter-sanity/   # @crudstack/adapter-sanity (Sanity.io implementation)
│   ├── adapter-firebase/ # @crudstack/adapter-firebase (Firebase implementation)
│   ├── ui/               # @crudstack/ui (shadcn components, layouts, views)
│   ├── auth/             # @crudstack/auth (Authentication providers)
│   ├── eslint-config/    # Shared ESLint configuration
│   └── typescript-config/# Shared TypeScript configuration
└── packages/cli/         # create-crudstack-app (The scaffolding CLI)
```

## 🚀 Quick Start

### Using the CLI (Recommended)

Once published, you can bootstrap a new project in seconds:

```bash
pnpm create crudstack-app my-app
cd my-app
pnpm dev
```

## 🔗 Usages

Here is how to quickly install and use the Firebase adapter for both Database and Authentication in your Next.js app.

### 1. Installation

Install the core package, the Firebase adapter, and the base Firebase SDK:

```bash
pnpm add @crudstack/core @crudstack/firebase firebase
```

### 2. Setup Firebase Instances

First, initialize your Firebase app and export the specific `auth` and `db` instances. This ensures Firebase is only initialized once, even with Next.js hot-reloads.

```ts
// lib/firebase.ts
import { initializeApp, getApps } from "firebase/app";
import { getAuth } from "firebase/auth";
import { getFirestore } from "firebase/firestore";

const firebaseConfig = {
    apiKey: process.env.NEXT_PUBLIC_FIREBASE_API_KEY,
    authDomain: process.env.NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN,
    projectId: process.env.NEXT_PUBLIC_FIREBASE_PROJECT_ID,
    storageBucket: process.env.NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET,
    messagingSenderId: process.env.NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID,
    appId: process.env.NEXT_PUBLIC_FIREBASE_APP_ID,
};

// Initialize Firebase safely
const app = !getApps().length ? initializeApp(firebaseConfig) : getApps()[0];

// Export the specific instances to be used by Crudstack
export const firebaseAuth = getAuth(app);
export const firebaseDb = getFirestore(app);
```

### 3. Initialize CrudStack (The Unified Hub)

Now, import those Firebase instances and pass them into the `FirebaseDatabaseAdapter` and `FirebaseAuthAdapter`.

```ts
// lib/crudstack.ts
import { CrudStack } from "@crudstack/core";
import { FirebaseDatabaseAdapter, FirebaseAuthAdapter } from "@crudstack/firebase";
import { firebaseDb, firebaseAuth } from "./firebase";

// Initialize the unified CrudStack instance
export const crudstack = new CrudStack({
    database: new FirebaseDatabaseAdapter({ db: firebaseDb }),
    auth: new FirebaseAuthAdapter({ auth: firebaseAuth }),
});
```

### 4. Database Usage (CRUD)

Use the `crudstack.db()` factory to create type-safe services for any collection.

```ts
// app/actions.ts
"use server";
import { projectService } from "@/lib/services/project";
import { revalidatePath } from "next/cache";

export async function createProject(data: Omit<Project, "id">) {
    // Create a new document in Firebase
    await projectService.create({ id: "", ...data });
    revalidatePath("/dashboard/projects");
}

export async function getProjects() {
    return await projectService.getList();
}
```

### 5. Auth Usage

Wrap your app with the `CrudStackProvider` and use the unified hooks.

```tsx
// app/layout.tsx
import { CrudStackProvider } from "@crudstack/core";
import { crudstack } from "@/lib/crudstack";

export default function RootLayout({ children }: { children: React.ReactNode }) {
    return (
        <html lang="en">
            <body>
                {/* Pass the unified instance to the Provider */}
                <CrudStackProvider value={crudstack}>{children}</CrudStackProvider>
            </body>
        </html>
    );
}
```

```tsx
// components/login.tsx
"use client";
import { useCrudStack } from "@crudstack/core";

export function LoginButton() {
    // Access both database and auth from the unified hook!
    const { db, auth } = useCrudStack();
    const { user, signInWithGoogle, signOut } = auth;

    return user ? (
        <button onClick={signOut}>Logout ({user.email})</button>
    ) : (
        <button onClick={signInWithGoogle}>Login with Google</button>
    );
}
```
